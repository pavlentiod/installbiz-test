# Сервис скачивания и анализа файлов

Легковесное тестовое приложение на FastAPI. Оно последовательно скачивает неизвестный
заранее каталог через API разработчиков, соблюдает `Retry-After`, сохраняет файлы в
SQLite через SQLAlchemy 2 async и считает частоту цифр в выбранных файлах.

## Возможности

- устойчивое последовательное скачивание группами не более трёх файлов;
- постоянный `X-Candidate-Id`;
- обработка `403`, `429`, сетевых ошибок и `5xx`;
- строгая проверка ZIP и строки из 500 цифр;
- восстановление незавершённого процесса после перезапуска;
- сортировка и пагинация скачанных файлов;
- выбор отдельных файлов, страницы или всего каталога;
- общая и пофайловая статистика цифр;
- Swagger UI на `/docs` и форматированные консольные логи.

## Структура

Код приложения разделён на тематические модули в `src`: конфигурацию, слой хранения,
загрузчик, HTTP-маршруты и сборку FastAPI-приложения. SQLAlchemy создаёт таблицы через
`Base.metadata.create_all()` при старте — Alembic для тестового приложения не
используется. Каталог `developer_api_client` сгенерирован
из `openapi.json` с помощью `openapi-python-client` и вручную не редактируется. HTML и
небольшой JavaScript находятся в `templates`.


## Состояния скачивания

- `idle` — ещё не запускалось;
- `queued` / `running` — процесс запущен;
- `waiting_retry` — приложение соблюдает паузу из `Retry-After`;
- `completed` — API вернул пустой список имён;
- `failed` — получена постоянная ошибка или исчерпан лимит сетевых повторов.

Сначала файлы проверяются и сохраняются в SQLite, и лишь затем отправляется
`POST /api/files/downloaded`. Повторное подтверждение безопасно по контракту API и
позволяет восстановиться после обрыва соединения.
